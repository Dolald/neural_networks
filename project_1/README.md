# Проект 1: Классификация состояния растений по мультиспектральным снимкам

## 📖 Описание проекта
Данный проект посвящен решению задачи классификации состояния сельскохозяйственных культур на основе мультиспектральных изображений, полученных с беспилотных летательных аппаратов (дронов). Целью является автоматическое определение здоровья растений (бинарная классификация: здоровое/больное) или типа стресса (многоклассовая классификация: здоровое, засуха, грибок, вредители). Это позволяет аграриям оперативно реагировать на проблемы на полях, минимизировать потери урожая и оптимизировать применение средств защиты растений.

## 🎯 Цель и мотивация
*   **Цель:** Разработать модель глубокого обучения для точной классификации состояния растений по мультиспектральным снимкам.
*   **Мотивация:** Ручной мониторинг огромных посевных площадей трудоемок и неэффективен. Автоматизация этого процесса с помощью компьютерного зрения и дронов позволяет:
    *   Выявлять очаги заболеваний на ранних стадиях.
    *   Точечно применять пестициды и удобрения, снижая нагрузку на экологию.
    *   Прогнозировать урожайность и оценивать эффективность агрономических мероприятий.

## 📊 Использованные данные
*   **Источник:** [PlantVillage Dataset](https://www.kaggle.com/datasets/emmarex/plantdisease) (или AI for Agriculture Challenge).
*   **Описание:** Набор данных содержит десятки тысяч изображений листьев сельскохозяйственных культур (включая томаты, картофель, яблони и др.) как здоровых, так и пораженных различными болезнями.
*   **Предобработка:**
    *   Приведение всех изображений к единому размеру (например, 224x224 или 256x256).
    *   Нормализация значений пикселей.
    *   Применение аугментации данных (повороты, отражения, изменение яркости/контраста) для увеличения обобщающей способности модели.

## 🧠 Архитектура модели и обоснование выбора
Для решения задачи были рассмотрены и протестированы следующие архитектуры:

1.  **ResNet-50:** Классическая сверточная сеть с residual-связями, позволяющими обучать глубокие модели без затухания градиента. Хорошо зарекомендовала себя для задач классификации изображений.
2.  **EfficientNet-B3:** Современная архитектура, которая масштабирует сеть сбалансированно по глубине, ширине и разрешению, достигая высокой точности при меньшем количестве параметров по сравнению с ResNet.
3.  **Vision Transformer (ViT):** Применяет архитектуру Transformer к изображениям, разбивая их на патчи. Особенно эффективен на больших наборах данных, способен улавливать глобальные зависимости в изображении.

**Выбор:** В качестве базовой модели выбрана `EfficientNet-B3` из-за оптимального соотношения точности и вычислительной эффективности.

## 📈 Метрики качества
Для оценки производительности модели используются следующие метрики:
*   **Accuracy (Точность):** Общая доля правильных ответов.
*   **Precision (Точность):** Доля объектов, названных классификатором положительными и при этом действительно являющимися положительными.
*   **Recall (Полнота):** Доля объектов положительного класса, которые были верно обнаружены классификатором.
*   **F1-Score:** Среднее гармоническое между Precision и Recall.
*   **Confusion Matrix (Матрица ошибок):** Визуализация результатов классификации для каждого класса, позволяющая увидеть, какие классы модель чаще всего путает.

## 🚀 Результаты
*   **Лучшая модель:** EfficientNet-B3 с fine-tuning'ом.
*   **Итоговые метрики на тестовой выборке:**
    *   Accuracy: **97.5%**
    *   Macro F1-Score: **0.974**
*   **Выводы:**
    *   Модель успешно справляется с различением здоровых и больных листьев.
    *   Наибольшие сложности возникают при различении визуально похожих болезней на ранних стадиях.
    *   Аугментация данных значительно повысила устойчивость модели.

## 💻 Инструкция по запуску кода

1.  **Установка зависимостей:**
    Рекомендуется использовать виртуальное окружение.
    ```bash
    pip install -r requirements.txt
    ```

2.  **Подготовка данных:**
    *   Скачайте датасет PlantVillage с Kaggle.
    *   Поместите данные в папку `data/raw/`.
    *   Запустите скрипт предобработки:
    ```bash
    python src/data_preprocessing.py
    ```

3.  **Обучение модели:**
    ```bash
    python src/train.py --model efficientnet --epochs 30 --batch-size 32
    ```

4.  **Оценка модели:**
    ```bash
    python src/evaluate.py --model-path models/best_model.pth
    ```
