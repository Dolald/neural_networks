# Проект 3: Система рекомендаций по удобрениям на основе состояния поля

## 📖 Описание проекта
Данный проект представляет собой систему поддержки принятия решений для агрономов. На основе информации о состоянии почвы (содержание NPK, pH), погодных условиях (температура, осадки) и данных о прошлом урожае, модель машинного обучения генерирует рекомендации по оптимальным дозам внесения азотных (N), фосфорных (P) и калийных (K) удобрений.

## 🎯 Цель и мотивация
*   **Цель:** Разработать модель multi-output регрессии для предсказания оптимальных доз NPK.
*   **Мотивация:**
    *   Чрезмерное внесение удобрений вредит экологии (эвтрофикация водоемов) и экономике хозяйства.
    *   Недостаточное внесение ведет к недобору урожая.
    *   Модель, учитывающая множество факторов, помогает найти баланс для достижения максимальной экономической эффективности и экологической устойчивости.
    *   В качестве метрики используется симулированная урожайность: сравнивается результат при использовании рекомендаций модели против базовых доз.

## 📊 Использованные данные
*   **Источник:** [FAOSTAT](https://www.fao.org/faostat/en/#data) / [Agricultural Yield and Fertilizer Use](https://www.kaggle.com/datasets/timmofeyy/agricultural-yield-and-fertilizer-use-dataset) (синтезированный или реальный датасет с Kaggle).
*   **Описание:** Датасет включает в себя записи по разным полям/регионам за несколько лет:
    *   **Входные признаки:** Тип почвы, pH, содержание N/P/K в почве до внесения, температура, осадки, прошлая урожайность, сорт культуры.
    *   **Выходные признаки:** Рекомендуемые дозы N, P, K.
    *   **Целевая метрика:** Фактически полученная урожайность при данных дозах.
*   **Симуляция:** Для оценки качества рекомендаций используется простая математическая модель зависимости урожайности от доз удобрений (например, закон убывающей отдачи Митчерлиха), параметры которой оценены по данным.

## 🧠 Архитектура модели и обоснование выбора
Для решения задачи предсказания трех целевых переменных (N, P, K) использованы два подхода:

1.  **Multi-Output MLP (Многослойный перцептрон):** Простая и эффективная архитектура. На вход подаются все признаки, на выходе — 3 нейрона с линейной активацией. Модель обучается минимизировать суммарную ошибку по всем трем выходам.
2.  **Gradient Boosting (XGBoost/ CatBoost):** Классический ансамблевый метод, отлично работающий с табличными данными. Для multi-output задачи используется стратегия "цепи регрессоров" (Regressor Chain) или отдельные модели для каждой целевой переменной.

**Выбор:** В качестве основной модели выбран **Multi-Output MLP**, так как он легко интегрируется в сквозной пайплайн обучения и может изучать скрытые зависимости между рекомендуемыми дозами разных элементов. XGBoost используется как сильный базовый метод (baseline).

## 📈 Метрики качества
1.  **MAE (Mean Absolute Error) для доз:** Оценивает точность предсказания самих доз N, P, K (кг/га).
2.  **Симулированная урожайность (Yield Gain):** Ключевая метрика. Сравнивается предсказанная урожайность (по модели-симулятору) при использовании:
    *   Рекомендаций модели (`yield_model`).
    *   Базовой стратегии (средние дозы по региону) (`yield_baseline`).
    *   Стратегии фермера (исторические данные) (`yield_actual`).
    *   Метрика: `(yield_model - yield_baseline) / yield_baseline * 100%` (относительный прирост урожая).

## 🚀 Результаты
*   **Точность предсказания доз (MAE):**
    *   N: 5.2 кг/га
    *   P: 3.1 кг/га
    *   K: 4.5 кг/га

*   **Эффективность рекомендаций (симуляция):**
    *   Модель рекомендует дозы, которые в среднем на **12%** превышают урожайность базовой стратегии (внесение средних доз).
    *   По сравнению с историческими данными фермеров, модель позволяет снизить расход азотных удобрений на **15%** без потери урожайности.

## 💻 Инструкция по запуску кода

1.  **Установка зависимостей:**
    ```bash
    pip install -r requirements.txt
    ```

2.  **Подготовка данных и симулятора:**
    ```bash
    python src/data_preparation.py
    python src/create_yield_simulator.py
    ```

3.  **Обучение модели:**
    ```bash
    python src/train.py --model mlp --epochs 100
    ```

4.  **Оценка и симуляция:**
    ```bash
    python src/evaluate_with_simulation.py --model-path models/best_mlp.pth
    ```
