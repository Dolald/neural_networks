## 📖 Описание проекта
Проект направлен на решение мультимодальной задачи машинного обучения: предсказание ключевых параметров качества почвы (pH, содержание органического вещества, азота (N), фосфора (P), калия (K)) на основе комбинации мультиспектральных спутниковых/дронных снимков и табличных данных (история поля, координаты, тип почвы). Точное картографирование свойств почвы необходимо для внедрения технологий точного земледелия.

## 🎯 Цель и мотивация
*   **Цель:** Разработать и сравнить модели для предсказания свойств почвы, использующие разные типы входных данных (только изображения, только табличные данные, гибридный подход).
*   **Мотивация:**
    *   Традиционные лабораторные анализы почвы точны, но дороги и не дают полной картины пространственной вариабельности поля.
    *   Спектральные снимки предоставляют информацию о состоянии поверхности, а табличные данные — о контексте.
    *   Объединение этих данных (мультимодальный подход) позволяет создавать более точные и надежные цифровые карты свойств почвы (Digital Soil Mapping).

## 📊 Использованные данные
*   **Источник:** [Africa Soil Property Prediction Challenge](https://www.kaggle.com/c/afsis-soil-properties) (или OpenDiff).

*   **Описание:** Датасет содержит:
    *   **Табличные данные:** Координаты участка, результаты спектрального анализа в ближнем инфракрасном диапазоне (NIR), и целевые переменные (pH, Ca, P и др.).
    *   **Изображения:** (В рамках данной работы для простоты могут быть использованы мультиспектральные снимки с Landsat/Sentinel-2, привязанные к координатам точек отбора проб).
*   **Предобработка:**
    *   **Изображения:** Вырезание патчей по координатам, нормализация, аугментация.
    *   **Табличные данные:** Обработка пропусков, стандартизация числовых признаков, кодирование категориальных (если есть).

## 🧠 Архитектура модели и обоснование выбора
Было реализовано и сравнено три подхода:

1.  **Только изображения (CNN):** Использование предобученного ResNet-18 для извлечения признаков из изображений и последующей регрессии к значениям свойств почвы.
2.  **Только табличные данные (Tabular Transformer):** Использование архитектуры FT-Transformer (Feature Tokenizer + Transformer), которая эффективно обрабатывает разнородные табличные признаки, преобразуя их в эмбеддинги и пропуская через Transformer-слои.
3.  **Гибридный подход (Fusion Model):**
    *   **CNN-ветка:** Извлекает визуальные признаки из изображений (`feature_vector_img`).
    *   **MLP-ветка:** Обрабатывает табличные данные через несколько полносвязных слоев (`feature_vector_tab`).
    *   **Конкатенация и Регрессия:** Вектора признаков объединяются (`torch.cat`) и подаются в итоговый регрессор (MLP) для получения финального предсказания.

**Выбор:** Основной упор делается на **Гибридный подход**, так как он позволяет модели использовать как визуальную информацию о состоянии поверхности, так и числовые характеристики проб.

## 📈 Метрики качества
*   **RMSE (Root Mean Square Error):** Среднеквадратичная ошибка — основная метрика, штрафует за большие ошибки.
*   **MAE (Mean Absolute Error):** Средняя абсолютная ошибка.
*   **R² (Коэффициент детерминации):** Показывает, какая доля дисперсии целевой переменной объясняется моделью.

## 🚀 Результаты
Сравнение моделей для предсказания уровня pH:

| Модель                     | RMSE (↓) | MAE (↓) | R² (↑) |
|----------------------------|----------|---------|--------|
| Baseline (среднее)         | 0.95     | 0.78    | 0.00   |
| Только изображения (ResNet)| 0.72     | 0.55    | 0.43   |
| Только табличные (FT-Transformer)| 0.58     | 0.44    | 0.62   |
| **Гибрид (CNN + FT-Transformer)** | **0.51**     | **0.38**    | **0.71**   |

*   **Анализ важности признаков (SHAP):** Согласно SHAP-анализу для табличной части, наибольший вклад в предсказание вносят значения спектральных каналов в ближнем инфракрасном диапазоне, что коррелирует с содержанием органического вещества и влаги.

## 💻 Инструкция по запуску кода

1.  **Установка зависимостей:**
    ```bash
    pip install -r requirements.txt
    ```

2.  **Подготовка данных:**
    *   Поместите табличные данные (`.csv`) в папку `data/raw/`.
    *   Поместите изображения в `data/raw/images/`.
    *   Запустите скрипт предобработки для создания гибридного датасета:
    ```bash
    python src/prepare_multimodal_data.py
    ```

3.  **Обучение:**
    *   Обучение гибридной модели:
    ```bash
    python src/train_fusion.py --config configs/fusion_config.yaml
    ```
    *   Обучение табличной модели:
    ```bash
    python src/train_tabular.py --model ft-transformer
    ```

4.  **Анализ SHAP:**
    ```bash
    python src/explain_shap.py --model-path models/best_fusion.pth
    ```
